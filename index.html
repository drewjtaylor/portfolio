Weeklydire

Full MERN app with user authentication

"WeeklyDire" is a fake news site (not to be confused with a fake-news site) to showcase a complete, cloud-deployed app inspired by overcoming tragedy and silver linings.

My most complicated and robust project to date, Weeklydire uses MongoDB for a database, Express/Node for the server, and React for the front end (MERN). This app tracks users, articles, comments, and administrator operations through the website.

To test out the full functionality, you can sign in with username: "testuser" and password: "testpassword", allowing you to see the "Write" and "Admin" links.

Formation Church

Static website updated with state management for events

"Formation Church" was one step closer to what the church founder visualized. Other than updating the name of the church from Formed Church to Formation Church, the website got an update as well!

In addition to trying a different visual style, this single-page-application introduced an interactive element in the form of an events calendar. The form to enter new events validates on the fly, and the event list displays all future events in date order. At this stage it uses dummy data, as it has not yet been connected to the back end. (Event dates were set out to 3023 on purpose so make sure they stay visible.)

Qwixxx Scoreboard

React/Redux state management

Qwixxx is a dice game where players score points based off of multiple dice rolls. Think Yahtzee-Plus.

The scoreboard was created using React, and the state is kept using Redux. Since Redux technically stores a NEW complete state object, as opposed to to updating an object, it was very easy to implement an undo/redo function.
